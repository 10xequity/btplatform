## 4. What v0.13.0 contains (functional)
- **Auto-claim:** generateSchedule (tournaments) + generateWeek (leagues) call
  facility.autoClaimForEvent — one `source='auto'` booking per event/week night; defaults VB 1..N,
  busy defaults move to next open courts ("moved to open areas" per owner), response reports
  courts/moved/shortfall/note; regenerate releases + re-claims; deleteWeek releases that date;
  claims NEVER block schedule generation (try/catch). Window: tournaments = starts_at time +
  budgetMinutes (420 default); leagues = starts_at time + weekMinutes (180 default), week N =
  starts_at + 7(N−1) days. Claims are normal bookings → drag/edit/delete in the facility UI.
- **Rental requests:** POST /api/rental-request (signed-in only) → pending; staff panel on
  admin-facility (preset picker, Approve conflict-checks then books under org 10 source='rental',
  Decline). **Public self-serve rental deliberately HIDDEN** — no member UI shipped yet by design.
