## Gotchas (learned the hard way)
- **Cloudflare MCP cannot deploy workers** (D1/KV/R2 + read-only worker tools only) — deploys go through the GitHub Actions workflow instead. DB work: do it directly via MCP.
- GitHub web-UI drag-drop upload can fail silently AND never uploads dotfolders — verify repo contents after any owner upload.
- GitHub MCP currently read-only on this repo (403 on push) until owner grants write.
- Pages runs Jekyll by default → `.nojekyll` required at repo root.
- D1 MCP queries: multi-statement SQL works; sqlite_master includes `_cf_*` tables — filter them.
- `standings` PK is (event_id, team_id) and rows upsert on every score — don't insert duplicates.
- Regenerating a schedule requires `confirm_wipe_scores:true` if any game is scored (safety I added — keep it).
- Frontend auth uses Bearer-token fallback in sessionStorage because Safari blocks third-party cookies between github.io and workers.dev.
- Owner has no coding background: click-by-click instructions, plain English, never ask for terminal use; secrets are pasted by the owner directly into GitHub/Cloudflare — never through chat.
- File naming: docs/migrations get `YYYY-MM-DD_name_vX.Y`; web-served files keep functional names with version headers inside; CHANGELOG entry per bump.
