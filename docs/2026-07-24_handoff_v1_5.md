## 4. What v0.12.0 contains (functional)
- **Model:** 13 courts (VB 1–13) + 6 rooms as atoms; presets reserve atom SETS (All Courts, Full
  Hardwood VB 1–8, Sports Court VB 9–13, Basketball Ct 1–4 overlays = VB pairs, Whole Facility).
- **Conflict engine:** date + time overlap + atom intersection = HARD; both sides Court Share →
  warning (`force:true` = "Book anyway (shared)"); closures always hard. Series (weekly, 52-wk cap)
  conflict-check every date BEFORE any write.
- **API (staff-gated):** /api/admin/facility/spaces · /bookings?from&to · POST /check ·
  POST /bookings · PATCH /bookings/:id (scope one|series=this+future) · DELETE ?scope= ·
  POST /import (header-mapped CSV, dry_run, per-line errors, hard-conflict rows skipped).
- **UI:** admin-facility.html — Day grid (spaces × 6:00–23:00, operator colors from
  orgs.brand_json.facility_color, hatched closures, today line) + Week list; booking modal with
  preset auto-check, share/closure toggles, weekly repeat, staffing/catering/door/POC/attendees
  details, inline conflict panel; CSV import with mandatory dry-run. Design system respected:
  no animation on grid nav (frequency rule), modal 200ms ease-out scale(.97), 44px, focus-visible.
